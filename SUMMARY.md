## Как работает система модулей

### Добавление нового модуля

**Шаг 1:** Создайте файл
```bash
nano modules/mymodule.js
```

**Шаг 2:** Добавьте код
```javascript
module.exports = {
  name: 'mymodule',
  version: '1.0.0',
  description: 'Мой модуль',
  enabled: true,
  
  async init(context) {
    this.bot = context.bot;
    console.log('✓ Модуль загружен');
  },
  
  commands: {
    mycommand: async function(msg) {
      await this.bot.sendMessage(msg.chat.id, 'Работает!');
    }
  }
};
```

**Шаг 3:** Перезагрузите бота
```bash
npm start
```

**Готово!** Модуль автоматически загружен и команда `/mycommand` работает.

### Что получает модуль при инициализации

```javascript
async init(context) {
  // 1. Экземпляр Telegram бота
  this.bot = context.bot;
  
  // 2. Хранилище данных модуля
  // Автоматически загружается из bot_data/modulename.json
  this.data = context.data;
  
  // 3. Функция сохранения
  // Сохраняет this.data в файл
  this.saveData = context.saveData;
}
```

### Жизненный цикл модуля

```
1. ЗАГРУЗКА
   ↓
   ModuleLoader сканирует modules/
   ↓
2. ВАЛИДАЦИЯ
   ↓
   Проверка name, version, description, enabled
   ↓
3. ИНИЦИАЛИЗАЦИЯ
   ↓
   Вызов init(context)
   ↓
4. РЕГИСТРАЦИЯ
   ↓
   Регистрация команд и обработчиков
   ↓
5. РАБОТА
   ↓
   Обработка команд и сообщений
   ↓
6. ВЫГРУЗКА (при необходимости)
   ↓
   Вызов destroy()
```
