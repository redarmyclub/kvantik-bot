# 🚀 Инструкция по установке Kvantik Bot v2.0

## Быстрая установка (5 минут)

### Шаг 1: Распаковка

```bash
# Распаковать архив
tar -xzf kvantik-bot-v2-fixed.tar.gz

# Перейти в директорию
cd kvantik-bot2
```

### Шаг 2: Установка зависимостей

```bash
npm install
```

### Шаг 3: Настройка

```bash
# Создать файл конфигурации
cp .env.example .env

# Отредактировать (используйте любой редактор)
nano .env
```

**Обязательно заполните:**
```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
MAIN_ADMIN_ID=ваш_telegram_id
```

**Как получить:**
- `TELEGRAM_BOT_TOKEN`: Напишите [@BotFather](https://t.me/BotFather) → /newbot
- `MAIN_ADMIN_ID`: Напишите [@userinfobot](https://t.me/userinfobot)

### Шаг 4: Запуск

```bash
npm start
```

Готово! Бот запущен ✅

## Возможные проблемы

### Ошибка: "Cannot find module"

**Решение:**
```bash
rm -rf node_modules
npm install
```

### Ошибка: "TELEGRAM_BOT_TOKEN is not defined"

**Решение:**
- Проверьте что файл `.env` существует
- Проверьте что `TELEGRAM_BOT_TOKEN` заполнен
- Перезапустите бота

### Бот не отвечает в Telegram

**Решение:**
- Проверьте что токен правильный
- Проверьте интернет-соединение
- Посмотрите логи: `tail -f bot_logs/errors.log`

### Модули не загружаются

**Решение:**
```bash
# Запустите проверку
node check.js

# Проверьте структуру модуля
# Должны быть поля: name, version, description, enabled
```

## Обновление

### С версии 1.x на 2.0

Все данные совместимы. Просто:

```bash
# 1. Сделайте бэкап данных
cp -r bot_data bot_data.backup

# 2. Распакуйте новую версию
tar -xzf kvantik-bot-v2-fixed.tar.gz

# 3. Скопируйте .env
cp старая_папка/.env kvantik-bot2/

# 4. Установите зависимости
cd kvantik-bot2
npm install

# 5. Запустите
npm start
```
