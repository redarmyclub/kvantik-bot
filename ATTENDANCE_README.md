## 📂 СТРУКТУРА EXCEL ФАЙЛА

Модуль работает с файлом, имеющим 3 листа:

### 1. Лист "Children"
- **Колонка A:** card_id (ID карты ребёнка)
- **Колонка B:** first_name (Имя)
- **Колонка C:** last_name (Фамилия)
- **Колонка G:** parent_phone (Телефон родителя) ← **ВАЖНО!**

### 2. Лист "Attendance_Log"
- **Колонка C:** card_id
- **Колонка D:** child_name (полное имя)
- **Колонка E:** action (CHECK_IN)
- **Колонка F:** time_in (время прихода)

### 3. Лист "Current_Session"
- **Колонка A:** card_id
- **Колонка B:** child_name
- **Колонка C:** check_in_time

## 🔧 РЕШЕНИЕ ПРОБЛЕМ

### ❌ Команды не работают

**Проблема:** Бот не отвечает на команды

**Решение:**
1. Убедитесь что вы отправляете от администратора
2. Проверьте что в `.env` правильный MAIN_ADMIN_ID
3. Используйте ПРЯМЫЕ слэши в пути: `/` не `\`

**Правильно:**
```
/attendance_path C:/Users/Виктор/Downloads/kvantik_data.xlsx
```

**Неправильно:**
```
/attendance_path C:\Users\Виктор\Downloads\kvantik_data.xlsx
```

### ❌ "Родитель не найден"

**Проблема:** В консоли пишет "Родитель не найден для телефона"

**Причина:** Телефон в Excel не совпадает с телефоном в боте

**Решение:**
1. Откройте `/attendance_status`
2. Посмотрите "Детей с телефоном"
3. Проверьте формат телефона в Excel (колонка G)
4. Убедитесь что родитель зарегистрирован в боте

### ❌ "Файл не найден"

**Проблема:** Модуль не может найти Excel файл

**Решение:**
- **Windows:** используйте прямые слэши `/`
- **Пример:** `C:/Users/Виктор/Downloads/file.xlsx`
- **Или:** двойные обратные `C:\\Users\\Виктор\\Downloads\\file.xlsx`

### ⚠️ Уведомления не приходят

**Проверьте:**
```
/attendance_status
```

Должно показывать:
- Мониторинг: ✅ Работает
- Файл найден: ✅ Да  
- Детей с телефоном: > 0

Если "Детей с телефоном: 0" - проверьте колонку G в листе Children!
